## 0.1 产品定位

我们要构建一个**多 Agent 金融研究报告自动生成系统**——用户输入一个 A 股股票代码（如 `600519`），系统自动完成数据采集、舆情分析、市场分析、报告撰写、合规审查，最终输出结构化的投资研究报告（PDF / Markdown / JSON）。

系统采用 **Pipeline + Fan-out 并行编排**模式：
- **Stage 1（Fan-out 并行）**：Data Collector 和 Sentiment Agent 同时执行，分别采集财务数据和舆情数据
- **Stage 2（Pipeline 串行）**：Market Analyst 等待 Stage 1 完成后综合分析
- **Stage 3（Pipeline 串行）**：Report Writer 生成结构化报告
- **Stage 4（Pipeline 串行）**：Compliance Agent 审查合规性

## 0.2 核心用户故事

| 角色 | 我想要 | 以便 |
|------|--------|------|
| 投资分析师 | 输入股票代码，让系统自动生成投资研究报告 | 节省手动检索、整理、分析的时间（人工需 4-6 小时，系统约 3 分钟） |
| 投资分析师 | 系统自动从多个数据源采集数据 | 无需逐一访问 AKShare、新闻网站、公告披露平台 |
| 合规审核员 | 报告生成后自动进行合规审查 | 确保报告符合监管要求，降低合规风险 |
| 技术负责人 | 系统可扩展，新增 Agent 或工具无需修改核心循环 | 降低维护成本，支持快速迭代 |

## 0.3 Agent 能力边界

- **能做的事**：
  - 从 AKShare 采集 A 股行情、财务指标、公告摘要
  - 从 Tavily 搜索并分析财经新闻的情感倾向
  - 计算技术指标（RSI、MA、波动率）并综合分析市场趋势
  - 自动生成包含 7 个章节的结构化投资研究报告
  - 自动审查报告合规性（免责声明、数据引用、监管合规、偏见检查）
  - 输出 PDF / Markdown / JSON 三种格式
- **不能做的事**：
  - 代替人类做最终投资决策
  - 实时交易或执行资金操作
  - 访问付费数据库（如 Bloomberg Terminal）

## 0.4 输入/输出契约

| 项目 | 说明 |
|------|------|
| 输入 | 股票代码（必填，如 `600519`）、分析周期（可选，默认 `1y`）、报告语言（可选，默认 `zh`） |
| 输出 | 在输出目录下生成 PDF + Markdown + JSON 三种格式的投资研究报告 |
| 数据模型 | Pydantic v2 强制 JSON Schema，确保 Agent 间数据一致性 |
